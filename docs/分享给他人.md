# 如何把 Labro 分享给同学使用

本应用是**本地运行**的工具：数据存在各自电脑上，不经过云端服务器（除 DeepSeek API 调用外）。

---

## 方式一：发压缩包（最简单）

### 你要做的

1. **不要**把 `backend/config.yaml`（含 API 密钥）打进压缩包。
2. 可删除以减小体积（对方会自动重装）：
   - `backend/.venv`
   - `frontend/node_modules`
   - `backend/data`（你的实验数据）
3. 右键项目文件夹 → 发送到压缩文件，发给对方（网盘/U盘均可）。

### 对方要做的

1. 安装 **Python 3.9+**、**Node.js 18+**（均勾选 Add to PATH）。
2. 解压后双击根目录 **`分享运行.bat`**。
3. 首次会提示编辑 `backend/config.yaml`，填入自己的 [DeepSeek API 密钥](https://platform.deepseek.com/)。
4. 再次双击 `分享运行.bat`，浏览器打开 **http://127.0.0.1:8000** 即可使用。

---

## 方式二：GitHub 分享

1. 将项目推到 GitHub **私有或公开仓库**。
2. 确认 `.gitignore` 已忽略 `backend/config.yaml`（已配置）。
3. 对方 `git clone` 后按「方式一」运行。

---

## 方式三：同一 WiFi 局域网共用（实验室一台电脑服务）

适合一台电脑开着，同组同学用手机/电脑访问。

1. 查本机 IP：PowerShell 输入 `ipconfig`，记下 IPv4（如 `192.168.1.5`）。
2. 编辑 `backend/config.yaml`：

```yaml
server:
  host: "0.0.0.0"   # 允许局域网访问
  port: 8000
```

3. 运行 `分享运行.bat`。
4. 他人浏览器访问：`http://192.168.1.5:8000`（换成你的 IP）。
5. Windows 防火墙若拦截，选择「允许访问」。

> 注意：共用一台机器时，实验数据都在该电脑的 `backend/data/` 里，不是每人一份。

---

## 方式四：开发者双窗口运行

与你自己开发时相同：

```powershell
# 终端 1
cd backend
.\.venv\Scripts\Activate.ps1
python run.py

# 终端 2
cd frontend
npm run dev
```

浏览器打开 http://localhost:5173

## 每人需要什么

| 项目 | 说明 |
|------|------|
| DeepSeek API Key | 每人用自己的账号，**不要共用你的密钥** |
| 网络 | 文献/图片 AI 解析需要联网；步骤查看与文字/数字录入可离线 |
| 图片导入 | 支持拍摄实验讲义、板书等照片，自动 OCR + DeepSeek 整理后生成步骤 |
| 系统 | Windows 10/11（脚本为 `.bat`）；Mac 需手动按 README 命令运行 |

## 常见问题

**Q：打开 8000 端口是白屏或 JSON？**  
A：未构建前端。请用 `分享运行.bat`，或先执行 `cd frontend && npm run build`。

**Q：AI 解析失败？**  
A：检查 `config.yaml` 里 `api_key`、网络与代理设置（见 README）。

**Q：能否做成手机 App？**  
A：当前是网页；手机浏览器访问局域网地址即可录入，后续可考虑 PWA。

---

## 安全提醒

- 勿将含 API 密钥的 `config.yaml` 上传公开网盘或 GitHub。
- 实验数据在本地 SQLite，分享电脑时注意备份或清理 `backend/data/`。
